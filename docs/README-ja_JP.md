# overengineered-train-game

## Java のデザインパターンを Python に無理やり移植する愚かさを示すサンプル

このリポジトリは、ボードゲーム「Ticket to Ride」の Python 実装を含んでいます。しかし、これは正しく動作する模範的なプロジェクト**ではありません**。**教育的な警告**として、Java のデザインパターンを Python に「丸写し」することがどれほど危険かを示すためのものです。

コードは極度にオーバーエンジニアリングされており、読みづらく、不要な抽象化が詰め込まれています。単純なアクションがコマンドパターンに、一行の条件分岐で済むカード効果がストラテジーパターンに、直接書けるオブジェクトがファクトリパターンに包まれ、さらには深い継承階層がデータフローの追跡をほぼ不可能にしています。本来なら数百行の clean な Python コードで済むはずだったものが、何十ものファイルにわたる数千行のコードに膨れ上がりました——すべての問題が Java OOP というハンマーのための釘に見えてしまったからです。

このプロジェクトは、Python を Java のように書くことが、どれほど肥大化して保守不可能なゴミを生むかを完璧に実証しています。生きたアンチパターンの博物館です。

## この酷いコードは誰が書いたのか？

**このソフトウェアアーキテクチャの設計は、私が行ったものではありません。**

このコードベースの大部分は、過去の「課題」（または「スプリント」）で、二人のチームメイトが主導して書いたものです。彼らは優れた Java 開発者です——大学の OOP コースがオブジェクト指向プログラミングをとてもよく教え込んでいるのです。しかし、うまくいきすぎるのはかえって悪いことだ。

そして、悪夢が始まった：彼らは Java の OOP 習慣をそのまま Python に適用しました。あらゆるアクションにクラスが必要。あらゆるバリエーションに抽象インターフェースが必要。あらゆるコレクションに独自のサブクラスが必要。そして彼らは「改善」を重ね続けました——より多くのパターン、より多くの抽象化、より多くのファイル——その結果、プロジェクトは読むのも嫌になるモンスターと化しました。私は恐怖を抱きながら、単純なカードゲームがまるでヤマタノオロチのように、無数の首と尾を生やしていく様をただ見守ることしかできませんでした。

## このリポジトリの目的

このリポジトリが存在する**唯一**の理由は、**「やってはいけないこと」を示すためです**。

- Python を Java のように書いてはいけない。

- すべてのものに対してクラスを作ってはいけない。

- デザインパターンを考えるなく適用してはいけない——まず「本当に必要か？」と自問すること。

- アーキテクチャの純粋さのために可読性を犠牲にしてはいけない。

もしあなたがオブジェクト指向プログラミングを学んでいる学生なら、これは学んだことを**どのように適用してはいけないか**を示す完璧な**反面教師**です。このコードを読み、警告のサインを認識することを学び、同じ過ちを犯さないでください。

このリポジトリは**教育と警告のためだけに存在します**。したがって、**コードに関する issue や pull request は一切受け付けません**。ただし、個人の学習や研究に利用することについては、全く問題ありません。

どうか、**絶対に**自分で「直そう」としないで、私に連絡しないでください。それでは本末転倒です。このプロジェクト全体は、アーキテクチャ設計の段階で深刻な問題を抱えており、教訓となることを目的としている。正しい実装を知りたいなら、**どうか他の場所を探してください**。あるいは、**このリポジトリごと AI（ChatGPT、Claude、Gemini、Mistral AI、DeepSeek、GLM、MiniMax など）に放り込んで、リファクタリングさせてください**。設計の選択について議論したいなら……おめでとうございます、あなたはすでに教訓を理解しています。

## 最後に

ディレクトリ構造を見てください。ファイルの数を数えてください。自分自身に問いかけてください：「Ticket to Ride」のクローンに、本当に `actions` フォルダ、`effects` フォルダ、`comps` フォルダ、`deck` フォルダ、`utils` フォルダ、そして重複したファイルがある `ui` フォルダが必要なのでしょうか？カードを引くという操作に、本当にコマンドパターンのクラスが必要なのでしょうか？カード効果に、本当にストラテジーパターンが必要なのでしょうか？

答えは、もちろん**ノー**です。
